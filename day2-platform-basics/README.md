# Day 2 - Salesforce Platform Basics

## 1. What is Salesforce Platform?

Salesforce Platform is a cloud-based CRM platform that helps businesses manage customers, sales, services, and applications. It provides tools for developers and admins to build applications using configuration and coding.

---

# 2. Explain App, Object, and Tab

## App
An App in Salesforce is a collection of tools, tabs, and objects grouped together for a specific business purpose.

Example:
Sales App, Service App, Hospital Management App.

## Object
Objects are database tables used to store data.

Examples:
Account, Contact, Opportunity, Patient, Doctor.

## Tab
A Tab is used to access objects and features in Salesforce UI.

Example:
Patient Tab opens Patient records.

---

# 3. Difference Between Configuration and Coding

| Configuration | Coding |
|---|---|
| No programming required | Requires programming |
| Uses clicks and settings | Uses Apex and code |
| Faster to implement | Used for complex logic |
| Example: Validation Rules | Example: Apex Trigger |

# 5. CRM Concepts in Salesforce

CRM concepts like Account, Contact, and Opportunity are stored as Salesforce Objects.

- Account → Company or Organization
- Contact → Customer or Person
- Opportunity → Sales Deal

These objects are grouped inside Apps for business processes.

---

# 6. Multi-Tenant Architecture

Multi-tenant architecture means multiple customers share the same Salesforce infrastructure while keeping their data secure and separate.
